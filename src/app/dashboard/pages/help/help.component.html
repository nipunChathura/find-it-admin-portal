Find It Admin Portal – user guide and section details

Overview

The Find It Admin Portal is the central place to manage your platform: admin users, customers, items, merchants, sub-merchants, categories, outlets, payments, and notifications. You must be logged in to use any section.

Layout

  • Sidebar (left): Main menu. Click Dashboard to see the home overview. User, Merchant, Category, Customer, and Help open directly. Item and Outlet expand to show sub-menus (e.g. Item → View items, Item Discount; Outlet → Outlet list, Outlet Schedule, Payment). The current page is highlighted in blue.
  • Top toolbar: App title Find It Admin Portal, your greeting (Hi, username), Notifications (bell icon with unread count), Account (profile icon), and Logout. The menu icon (top-left) collapses or expands the sidebar.
 dashboard Dashboard

What it is

Clicking Dashboard in the sidebar (or opening the portal after login) shows the Dashboard home page. This is the default start page and gives an overview of the platform.

Dashboard home sections

  • Welcome: Greeting with your username.
  • KPI stat cards: Users, Merchants, Sub Merchants, Items, Customers, Outlets, Categories. Each card shows a count from the API and links to the related section when clicked. Below that: Pending approvals and Active discounts.
  • Monthly income: Horizontal bar chart showing income (LKR) by month for the last 6 months. Data comes from the dashboard monthly-income API.
  • Recent notifications: Up to 5 latest notifications. Click View all notifications to open the full Notifications page.
  • Merchant summary: Total merchants and total outlets from the merchant-summary API.
  • Outlet distribution: Pie chart by category/merchant from the same API.
  • Quick actions: Buttons to go to User page, Merchants, Payments, and Discounts quickly.
 people User

What it is

User (sidebar) opens the Manage User page. Here you view and manage admin users who can log in to this portal. You can add new admins, edit details, change status, approve pending users, or delete users. From the Dashboard home, the User KPI card or the User quick action also navigates to this page.

Page layout

  • Top: Page title and Add User button.
  • Filters: Status dropdown (All, ACTIVE, INACTIVE, PENDING, APPROVED, REJECTED, FORGOT_PASSWORD_PENDING) and Search box. Click Search to apply; Clear resets and reloads the list.
  • Table: Columns: ID, Name/Username, Email, Status, Created Date, and Actions (Edit, Delete; for System Admin, Approve/Reject for PENDING users). Sort by column headers; use the paginator for page size (e.g. 5, 10, 25, 50) and page navigation.

How to add a user

  1. Click Add User.
  2. Enter Name, Username, Email, Password, Role, and Status (if your role allows).
  3. Click Save. The new user appears in the table and can log in with the username and password you set.

How to edit a user

  1. Find the user and click the Edit (pencil) icon in the Actions column.
  2. Update Username, Email, Role, or Status in the dialog. Only users with the right role can change status.
  3. Click Update to save.

Other actions

  • Delete: Click the delete (trash) icon and confirm. The user’s status is set to deleted; they can no longer log in.
  • Approve / Reject: As System Admin, for users with status PENDING you see Approve (check) and Reject. Approve or Reject (with optional reason) to update their status.
 person Customer

What it is

Manage Customer shows all customers (end users of the Find It platform). Customers are loaded from the API and can be filtered, searched, edited, or deleted.

Page layout

  • Filters: Status (All, ACTIVE, INACTIVE, PENDING), Membership (All, SILVER, GOLD, PLATINUM), and Search. Click Search to apply; Clear resets and reloads.
  • Table: Columns are ID, Name, Email, Phone, Membership, Status, and Actions (Edit, Delete). Sort by column header; use the paginator for page size and navigation.

How to edit a customer

  1. Click the Edit (pencil) icon on the row.
  2. In the dialog, update First name, Last name, Email, Phone number, NIC, Date of birth, Gender, Country, Membership, or Status.
  3. Click Update. Changes are sent to the backend.

How to delete a customer

Click the Delete (trash) icon, then confirm in the dialog. The customer is removed from the list (backend delete).

 inventory_2 Item

What it is

The Item menu has two sub-pages: Item (product/service catalogue) and Item Discount (discounts applied to items).

Item (catalogue)

  • View all items in a table. Use Search and Status / Availability filters to find items.
  • Add new items, Edit (pencil) or Delete (trash) from the Actions column. Sort and paginate as needed.
  • Items are typically linked to categories and merchants and can be used in offers and discounts.

Item Discount

  • Manage discounts that apply to specific items. Filter by Status and Item.
  • Use Add to create a discount (e.g. name, type, value, item, validity). Edit or Delete from the Actions column.
 store Merchant

What it is

Merchant is for managing merchants (businesses) and sub-merchants that use the Find It platform. Merchants can have sub-merchants and one or more outlets; they sell items. The Dashboard home shows total Merchants and Sub Merchants in KPI cards; both link to this page.

Page layout

  • Table: Lists merchants (and sub-merchants where applicable) with columns such as ID, Name, Email, Status, Parent merchant (for sub-merchants), and Actions.
  • Use Search and Status (and any other filters) to find merchants. Sort by column headers; use the paginator for paging.

What you can do

  • Add Merchant: Create a new main merchant or sub-merchant (name, email, address, contact, profile image, type, etc. as per the form).
  • Edit: Change merchant details from the Actions column (pencil icon).
  • Delete: Remove a merchant after confirming in the dialog. This may affect linked outlets and items depending on backend rules.
  • Approve / Reject: For pending merchants (or sub-merchants), System Admin can Approve or Reject with a reason.
 category Category

What it is

Category lets you manage categories used to group items (e.g. Food, Electronics). Categories help customers and admins find and filter items. The Dashboard home shows total Categories in a KPI card that links here.

Page layout

  • Table: Lists categories with columns such as ID, Name, Description, and Actions. Use Search and any filters to find categories; sort by column; use the paginator for paging.

What you can do

  • Add Category: Create a new category (name, description, etc.).
  • Edit: Update a category’s name or other fields from the Actions column.
  • Delete: Remove a category. Ensure no items depend on it, or reassign them first, to avoid errors.
 payment Payment

What it is

Payment is for managing payment methods or configurations linked to outlets. Open it from the main menu (Payment) or from Outlet → Payment. The Dashboard home has a Payments quick action that links here.

Page layout

  • Table: Lists payments with columns such as Outlet, Type, Status, and Actions. Use Outlet and Status filters and Search; sort and paginate as needed.

What you can do

  • Add: Create a new payment method/configuration for an outlet (type, account details, status).
  • Edit: Change payment details or status from the Actions column.
  • Delete: Remove a payment configuration after confirming in the dialog.
 notifications Notifications

What it is

Notifications keep you updated on events (e.g. new signups, system alerts). You can see them in a dropdown from the bell icon in the toolbar, or on the full Notifications page.

Bell icon (toolbar)

  • Click the bell in the top toolbar to open a dropdown panel.
  • A red badge on the bell shows the count of unread notifications (hidden when zero).
  • In the dropdown: each item shows title, message, time (e.g. “5m ago”), and type (info, success, warning, error). Unread items are highlighted.
  • Use the checkmark on a row to mark as read, or Mark all as read at the top.
  • Click View All to open the full Notifications page. The list is refreshed automatically (e.g. every 5 minutes).

Dashboard – Recent notifications

On the Dashboard home, the Recent notifications card shows the latest 5 notifications. Click View all notifications to navigate to the full Notifications page (same as sidebar Notifications).

Notifications page (full list)

  • Open from the bell dropdown (View All), from the sidebar (Notifications), or from Dashboard home (View all notifications).
  • Shows all notifications. You can Mark all as read or mark individual items as read.
 storefront Outlet

What it is

Outlet represents a physical or logical location (e.g. a store branch). Under the Outlet menu there are three sub-pages: Outlet, Outlet Schedule, and Payment. The Dashboard home shows total Outlets in a KPI card and an Outlet distribution pie chart (by category/merchant); both use data from the merchant-summary API.

Outlet (list)

  • View all outlets. Add new outlets, edit (name, address, merchant, etc.), or delete. Use search and filters to find specific outlets.

Outlet Schedule

  • Configure opening hours and schedules for each outlet (e.g. Mon–Fri 9–5, Sat 10–2). Add, edit, or remove schedule entries so customers see when outlets are open.

Payment

  • Same as the main Payment menu: manage payment methods/configurations for outlets.

Tips

  • Sidebar: Use the menu icon (top-left) to collapse or expand the sidebar for more content space.
  • Finding data: On list pages (User, Customer, Item, Merchant, etc.), use Search and filters (Status, dropdowns), then click Search or press Enter. Use Clear to reset filters.
  • Sorting and paging: Click column headers to sort. Use the paginator at the bottom to set page size (5, 10, 25, 50) and move between pages.
  • Dashboard data: KPI counts, monthly income, merchant summary, and outlet distribution come from the dashboard APIs. If a section shows zero or dummy data, check that the backend endpoints (e.g. /dashboard/summary, /dashboard/monthly-income, /dashboard/merchant-summary) are available and you are logged in.
  • Account: Click the Account (profile) icon in the toolbar to open your profile. You can view your username, role, and profile image, and use Change profile image or Change password.
  • Logout: Use Logout when you finish to keep your account secure.
